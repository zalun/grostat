## Design Context

### Users
Solar PV system owner monitoring a Growatt 10000 TL3-S inverter. Technical enough to use CLI tools and understand electrical parameters (voltage, power, energy), but the statistics window should be approachable and enjoyable, not intimidating. Primary use: checking daily/monthly production and comparing periods to spot trends or issues (like grid voltage problems).

### Brand Personality
**Minimal, curious, precise.** The app should feel like a well-crafted instrument — clean and elegant but with depth when you dig in. Not a flashy dashboard, not a dry spreadsheet.

### Aesthetic Direction
- **Visual tone:** Minimal and elegant with moments of delight. Lots of whitespace, thin typography, subtle color accents. Data should breathe, not crowd.
- **References:** Google Analytics (line chart overlays, period comparison UX, summary cards) + Apple Health (colorful but restrained charts, system-native feel, inviting exploration)
- **Anti-references:** Grafana (too dense, too technical), Bloomberg (too dark/overwhelming), generic Bootstrap dashboards
- **Theme:** Follow macOS system (dark/light). Charts should look great in both modes.

### Color System
- **Primary accent:** Solar amber/gold — evokes energy, sun, production
- **Secondary accent:** Cool blue — for comparison period lines, secondary data
- **Semantic colors:** Green (normal voltage), Orange (warning >=250V), Red (critical >=253V) — consistent with existing menu bar alerts
- **Chart palette:** Primary line in accent gold, comparison line in muted blue/gray, area fills at low opacity
- **Backgrounds:** System default (`.background`), cards with subtle elevation

### Typography
- System font (SF Pro) throughout — native macOS feel
- Monospaced digits for data values (`.monospacedDigit`)
- Large, bold numbers for summary stats (hero metrics)
- Section headers in uppercase caption, consistent with existing popover

### Charts
- Line charts as primary visualization (not bars)
- Two lines overlaid for comparison: primary period solid + accent color, comparison period dashed + muted
- Subtle grid lines, minimal axis labels
- Interactive: hover/click for tooltip with both period values
- Smooth curves, not angular — SwiftUI Charts `.interpolationMethod(.catmullRom)`
- Area fill under primary line at ~10% opacity for visual weight

### Layout Principles
- Maximum two charts visible at once
- Summary cards above charts (hero metrics: total energy, peak power, delta %)
- Generous padding (16-20pt)
- Clear visual hierarchy: period selector > summary > charts > details

### Design Principles
1. **Data should delight** — Make numbers beautiful and explorable, not just informative
2. **Comparison is the story** — Every view should naturally support "this vs that" thinking
3. **Breathe** — Generous spacing, never cramped. Two charts max, not six.
4. **System-native** — Look and feel like a first-party macOS app, not an Electron wrapper
5. **Progressive disclosure** — Simple at a glance, detailed on interaction (tooltips, drill-down)
